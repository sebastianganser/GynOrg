🧪 Test: useHolidays Multi-Jahr Support

Dieser Test validiert die erweiterte useHolidays Hook mit Jahr-Parameter und Caching.

📋 Test-Szenarien

✅ Implementierte Features

  • Jahr-Parameter: useHolidays(year?, additionalFilter?)
  • Jahr-basiertes Caching: 5-Minuten Cache pro Jahr und Kalendereinstellungen
  • Automatische Jahr-Erkennung: Fallback auf aktuelles Jahr
  • Erweiterte Return-Werte: currentYear, availableYears, setYear
  • Performance-Optimierungen: Cache-basierte Wiederverwendung
  • Rückwärtskompatibilität: useHolidaysLegacy für alte API

🎯 Neue API-Verwendung

// Neue, verbesserte API
const { 
  holidays, 
  loading, 
  error, 
  currentYear, 
  availableYears, 
  setYear 
} = useHolidays(2024);

// Jahr wechseln
setYear(2025);

// Mit zusätzlichen Filtern
const { holidays } = useHolidays(2024, { 
  month: 12, 
  federal_state: 'BY' 
});

🔄 Rückwärtskompatibilität

// Alte API funktioniert weiterhin
const { holidays, loading, error } = useHolidaysLegacy({ 
  year: 2024, 
  month: 12 
});
