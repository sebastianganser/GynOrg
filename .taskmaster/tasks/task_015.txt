# Task ID: 15
# Title: Implement Desktop-Optimized Split-Screen Login Layout
# Status: done
# Dependencies: 3, 14
# Priority: medium
# Description: Redesign the current centered, card-based login page into a desktop-optimized split-screen layout, with the login form on the left (40-50% width) and a branding/info section on the right (50-60% width), specifically targeting desktop browsers while maintaining mobile responsiveness.
# Details:
Refactor LoginForm.tsx to implement a split-screen layout using Tailwind CSS with Flexbox or CSS Grid. On desktop (Chrome/Firefox), the left panel (40-50% width) should contain the login form, while the right panel (50-60% width) displays branding, imagery, or informational content. Use Tailwind utility classes to ensure a clean, modern look and professional spacing. The layout should gracefully degrade to a single-column, centered card on mobile devices. Ensure accessibility (contrast, keyboard navigation) and test for cross-browser consistency. Reference best practices from modern split-screen login designs, such as SeedProd and OptinMonster, which use a distraction-free form area and a visually engaging side panel for branding or messaging. Avoid unnecessary distractions and keep the login process clear and prominent. Update any related styles or assets as needed.

# Test Strategy:
- On desktop browsers (Chrome, Firefox), verify that the login page displays as a split-screen with correct proportions (login form left, branding/info right).
- Confirm the login form is fully functional and visually prominent.
- Check that the branding/info section displays correctly and does not interfere with the login process.
- Resize the browser window and test on mobile devices to ensure the layout reverts to a centered, card-based design with no split-screen.
- Test keyboard navigation and screen reader accessibility for both panels.
- Validate visual consistency and responsiveness across supported browsers.
