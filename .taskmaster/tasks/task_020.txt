# Task ID: 20
# Title: Implement Comprehensive School Holidays Integration for All 16 German Federal States with Dual-API Support and UI Enhancements
# Status: pending
# Dependencies: 18, 19
# Priority: high
# Description: Develop a dual-API system integrating ferien-api.de for 2023-2025 and mehr-schulferien.de for 2026 onwards, extend calendar settings to include federal state selection for school holidays, enhance the absence calendar frontend to distinguish holidays and school vacations, and provide an admin interface for managing school holidays.
# Details:
1. Backend Integration:
- Implement a service module to fetch school holiday data from ferien-api.de for years 2023-2025 using their REST API (e.g., https://ferien-api.de/api/v1/holidays/{stateCode}/{year}).
- Implement a second service module to fetch school holiday data from mehr-schulferien.de API v2.1 for years 2026 and beyond (e.g., https://www.mehr-schulferien.de/api/v2.1/federal-states/{state}/periods?year={year}).
- Design logic to switch between APIs based on the year requested.
- Map all 16 German federal states to their respective codes for both APIs.
- Store and update holiday data in the database, ensuring consistency and multi-year support.

2. Calendar Settings Extension:
- Add a federal state selection dropdown in calendar settings to allow users to specify their state for school holiday display.
- Persist user selections and apply them to holiday data queries.

3. Frontend Integration:
- Modify the absence calendar UI to visually differentiate between public holidays and school holidays (e.g., different colors or icons).
- Ensure school holidays are displayed according to the selected federal state and year.
- Provide tooltips or details on hover/click for holiday type and name.

4. Admin Interface:
- Create an admin panel section for managing school holidays, including viewing imported data, triggering manual refreshes from both APIs, and resolving conflicts.
- Provide logs or status indicators for API data fetch success or failure.

5. Performance and Caching:
- Implement caching strategies for API responses to minimize external calls.
- Ensure data is refreshed periodically and on-demand via admin interface.

6. Documentation:
- Document the dual-API integration approach, configuration options, and UI changes for future maintenance.

Code examples:
- Use asynchronous HTTP clients for API calls.
- Example pseudocode for API switching:

```typescript
function fetchSchoolHolidays(stateCode: string, year: number) {
  if (year >= 2026) {
    return fetchMehrSchulferienHolidays(stateCode, year);
  } else {
    return fetchFerienApiHolidays(stateCode, year);
  }
}
```

- Frontend: Extend calendar component to accept holiday type and render accordingly.

Considerations:
- Handle API rate limits and failures gracefully.
- Validate and normalize data from both APIs to a common internal format.
- Ensure backward compatibility with existing holiday data.
- Support localization and accessibility in UI components.

3. UI/UX Tests:

4. Performance Tests:

5. Regression Tests:

6. User Acceptance Testing:

Automated test suites should cover backend services, API clients, frontend components, and admin interface.
